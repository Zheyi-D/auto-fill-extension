# resume-init — 简历自动填充初始化

## 前置条件

用户需要已经安装了「简历自动填充助手」Chrome 扩展（即本项目），且 `data.js` 在当前工作目录或用户指定了路径。

## 工作流程

### Step 1: 确认参数

向用户确认：
- 简历文件路径（必需）
- 扩展项目路径（默认为当前目录下的 `data.js`，或用户指定）

### Step 2: 读取简历

用合适的工具读取简历文件：
- `.docx`：用 Python 的 `docx` 库或 `unzip -p file.docx word/document.xml` 提取文本
- `.pdf`：用 Read 工具直接读取
- `.txt`：直接读取

### Step 3: 解析并提取信息

从简历文本中提取以下结构化信息：

| 分类 | 提取内容 |
|------|---------|
| 基本信息 | 姓名、英文名、邮箱、电话、所在地、求职意向 |
| 教育背景 | 硕/本科→学校全称、学位、专业、GPA、荣誉、起止时间 |
| 实习经历 | 每段→公司全称、职位、起止时间、2-3条核心描述（每条50-100字） |
| 项目经历 | 每个→项目名称、描述、成果/数据 |
| 校园经历 | 项目名、描述、成果（可选） |
| 技能 & 其他 | 语言、工具、技术栈、证书 |

### Step 3: 核对信息

向用户展示提取结果摘要，逐分类列出字段数和每段经历名称。用户确认无误后再继续。

### Step 4: 写入 data.js

修改项目根目录的 `data.js` 文件：

1. 保持 `DEFAULT_DATA` 的完整 JSON 结构不变
2. 逐字段替换 value 为真实信息
3. 格式要求：
   - 每段实习/项目只用**一个描述字段**（label 为"描述1""描述2"等），多条要点用 `\n` 换行合并
   - 每条信息 `{ label: "公司1", value: "XX科技" }` 这种形式
4. 如果简历中有 data.js 中不存在的分类，新增 category
5. 如果 data.js 中有简历里不存在的分类（如校园经历），删除该 category

### Step 5: 告知用户

完成后向用户报告：
- 更新了哪些 category、各多少个字段
- 新增/删除了哪些分类
- 下一步操作：在 Chrome `chrome://extensions/` 页面点「重新加载」
