# 简历自动填充助手 — Skill

## 一件事

帮用户装好 Chrome 扩展 + 把简历数据填进去。

用户说一句"我的简历在 xxx，帮我装那个自动填充插件"，你走完下面五步。

## Step 1: 获取代码

```bash
git clone https://github.com/Zheyi-D/auto-fill-extension.git
```

没有 git 就提示用户下载 ZIP，或直接在当前目录创建文件。

## Step 2: 读取简历

用户提供的简历文件（`.docx` / `.pdf` / `.txt`）。**有什么提取什么，没有就跳过：**

| 分类 | 提取字段 | 必需？ |
|------|---------|------|
| 基本信息 | 姓名、邮箱、电话、所在地、求职意向 | ✅ 必填 |
| 教育背景 | 学校全称、学位、专业、GPA、荣誉、起止时间 | ✅ 必填 |
| 实习经历 | 每段→公司、职位、起止时间、2-3 条描述 | 如有 |
| 项目经历 | 每个→名称、描述、成果 | 如有 |
| 校园/竞赛/论文 | 名称、描述 | 如有 |
| 技能 & 其他 | 语言、工具、技术栈 | 如有 |

## Step 3: 核对信息（重要，不要跳过）

向用户展示提取结果摘要，逐分类列出，**等待用户确认后再写入**：

```
📋 我从你的简历中提取到以下信息：

👤 基本信息（X 条）：姓名=张三、邮箱=xxx@email.com、...
🎓 教育背景（X 条）：硕士=XX大学 计算机科学、本科=XX大学 软件工程、...
💼 实习经历（N 段）：
  - 第1段：XX科技有限公司 / 产品实习生 / 20XX/03-20XX/06（2条描述）
  - 第2段：XX集团 / AI产品实习生 / 20XX/09-20XX/03（2条描述）
🚀 项目经历（N 个）：XX审核系统、企业智能问答Agent
🛠️ 技能 & 其他（X 条）

📌 以下分类因简历中没有对应内容，将被删除：[校园经历]

需要我调整什么吗？确认无误后我写入插件。
```

**用户确认后才执行 Step 4**。如果用户要求修改，调整后再次展示直到用户满意。

## Step 4: 写入 data.js

用户确认无误后，修改 `auto-fill-extension/data.js` 中的 `DEFAULT_DATA`：

- 逐条替换 fields 的 value 为用户真实信息
- 用户简历中**有的分类**保留并填真实数据，**没有的分类直接删掉整个 category**
- 每段实习/项目只用**一个描述字段**（label 为"描述1""描述2"等），多条要点用 `\n` 换行合并
- JSON 结构不要改，只改 fields 数组内容

## Step 5: 告知用户

- 更新了哪些分类、各多少字段
- 最后一步：Chrome → `chrome://extensions/` → 开发者模式 → 加载已解压 → 选 `auto-fill-extension/` 文件夹
